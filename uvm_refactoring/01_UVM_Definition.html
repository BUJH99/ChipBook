Verification Methodology

UVM Universal Verification Methodology

반도체 검증을 위한 재사용 가능한 표준 검증 프레임워크 (Reusable Verification Framework)입니다. SystemVerilog의 객체 지향 프로그래밍(OOP) 특성을 활용하여 검증 환경의 모듈화, 자동화 및 재사용성을 극대화합니다.

왜 UVM이 필요한가?

규모가 커지는 최신 반도체 설계에서 기존 검증 방식이 가진 한계를 극복하기 위해 등장했습니다.

일반 SystemVerilog Testbench의 한계

  • • 구조의 표준화 부재로 인해 엔지니어마다 코딩 스타일 파편화
  • • 컴포넌트 간 강한 결합(Hardcoded)으로 인한 스파게티 코드
  • • 다른 프로젝트나 블록(Block) 레벨 검증으로의 재사용 어려움

UVM Framework

  • • OOP 기반의 강력한 표준 아키텍처 제공
  • • 독립적인 모듈화(Agent, Env) 및 계층 구조 지원
  • • VIP(Verification IP) 형태로 극대화된 재사용성

UVM의 핵심 목적

1. 재사용성 (Reusability)

블록 레벨에서 검증된 컴포넌트(Agent 등)를 수정 없이 시스템 레벨(SoC) 환경으로 재사용할 수 있습니다.

2. 계층 구조 (Hierarchy)

Test, Env, Agent 단위로 명확한 계층 트리를 형성하여 테스트벤치의 구조를 일관되게 유지합니다.

3. Transaction 기반 검증

핀(Pin) 레벨의 복잡한 신호 제어 대신, 데이터 패킷(Transaction) 단위로 통신하여 코딩을 추상화합니다.

4. 자동화 검증

랜덤 테스트 생성(Randomization)과 자동 데이터 비교(Scoreboard)를 통해 사람의 개입을 최소화합니다.

5. Coverage 기반 완료 판단

설계 스펙이 모두 테스트되었는지 정량적인 수치(Functional Coverage)로 측정하여 검증 종료 시점을 명확히 합니다.
