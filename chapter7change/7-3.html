UVM Factory: 런타임 다형성 (Dynamic Type Replacement)

검증 환경(TB)의 소스 코드를 단 한 줄도 수정하지 않고, Test 최상단에서 set_type_override()를 호출하는 것만으로 특정 컴포넌트나 트랜잭션을 완전히 다른 클래스로 교체하여 생성합니다.

Registry Type Router
// 1. 클래스를 Factory에 등록 (Macro) class base_driver extends uvm_driver; `uvm_component_utils(base_driver) endclass class err_inj_drv extends base_driver; `uvm_component_utils(err_inj_drv) endclass
// 2. Env: new() 대신 create() 호출 drv = base_driver::type_id::create("drv", this);
Top Level
// 3. Test: Override 지시 (원본수정 없음) function void build_phase(uvm_phase phase); base_driver::type_id::set_type_override( err_inj_drv::get_type() ); endfunction

Phase 메커니즘과 Objection (시간/종료 제어)

시뮬레이션 시간이 소모되는 run_phase에서, 컴포넌트들이 Objection(이의 제기)을 하나라도 들고 있으면 다음 Phase로 넘어가지 않고 대기(Lock)하며, 총합이 0이 되는 순간 엔진은 즉시 종료됩니다.

실무 팁: Objection Kill

모니터나 드라이버에 forever 루프가 있어도, 전체 시스템의 Objection 총합이 0이 되면 UVM 엔진은 해당 Task들의 실행을 강제 종료(Kill)하고 즉시 다음 Phase(extract)로 이동합니다.

uvm_config_db: 계층 독립적 데이터 라우팅

깊은 계층 구조를 가로질러 Virtual Interface나 설정값을 전달하기 위한 중앙 집중형 해시맵(DB)입니다. 와일드카드(*)를 활용한 스코프(Scope) 문자열 매칭으로 데이터를 안전하게 배달합니다.

Hierarchical Data Query Flow
// tb_top.sv (최상단 연결 및 Set) my_if vif_inst(clk, rst_n); initial begin // <타입>::set(Context, Scope, Key, Value) // '*'를 사용하여 하위 모든 경로 접근 허용 uvm_config_db#(virtual my_if)::set( null, "uvm_test_top.env.agent.*", "vif", vif_inst ); end
Leaf Level
// driver.sv (하위 컴포넌트 Get) virtual my_if vif; function void build_phase(uvm_phase phase); // 자신의 계층(this)을 기반으로 "vif" 검색 if (!uvm_config_db#(virtual my_if)::get( this, "", "vif", vif)) begin `uvm_fatal("NOVIF", "Virtual interface not found!") end endfunction

스레드 동기화: uvm_barrier

병렬로 실행되는 여러 스레드(ex: Virtual Sequence 내 다중 인터페이스 제어)가 특정 지점에서 모두 만나야만 다음 단계를 진행할 수 있도록 통제하는 동기화 게이트입니다. 설정된 Threshold 만큼 wait_for()가 누적되면 일제히 블로킹이 풀립니다.

Barrier Synchronization (Threshold = 3)
// Virtual Sequence: 동기화 객체 초기화 uvm_barrier b; task body(); // 3개의 스레드를 대기시킬 배리어 생성 b = new("sync_barrier", 3); fork do_ahb_seq(b); // Thread 1 do_apb_seq(b); // Thread 2 do_axi_seq(b); // Thread 3 join endtask
// 각 개별 시퀀스 내부 동작 task do_ahb_seq(uvm_barrier b); // 1. 사전 초기화 (실행 속도 모두 다름) init_registers(); // 2. 다른 스레드들이 모두 올 때까지 대기 b.wait_for(); // 3. Threshold(3) 달성 시 일제히 해제되어 실행 drive_main_traffic(); endtask
