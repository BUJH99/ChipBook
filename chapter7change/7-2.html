💎 UVM 핵심 철학 (Reuse)

Test와 Env를 분리하여, 향후 DUT가 변경되더라도 구축된 검증 환경(Env) 코드는 수정 없이 그대로 재사용할 수 있습니다.

2

UVM 라이브러리 및 클래스 구성

my_test.sv
// 1. UVM Macros & Package Import (필수)
`include "uvm_macros.svh"
import uvm_pkg::*;

// 2. UVM Test Class Declaration
class my_test extends uvm_test;
`uvm_component_utils(my_test) // Factory Registration

my_env env;

virtual function void build_phase(uvm_phase phase);
super.build_phase(phase);
// Object creation using Factory 'create'
env = my_env::type_id::create("env", this);
endfunction

endclass : my_test
import uvm_pkg::*;

UVM 패키지 로드

UVM Base 클래스와 매크로를 사용하기 위해 파일 최상단에 선언해야 합니다.

`uvm_component_utils

Factory 자동 등록

클래스를 UVM Factory에 등록(Registration)하여 문자열 이름으로 동적 객체 생성이 가능하도록 만듭니다.

type_id::create()

객체 생성의 표준

SV의 new() 대신 create()를 사용하여, 나중에 다른 클래스로 가로채기(Override)가 가능하게 만듭니다.

3

시뮬레이션 엔진 기동: run_test()

1

run_test() 호출

최상위 initial 블록에서 호출되어 UVM 엔진을 깨웁니다.

2

Factory 동적 생성

UVM Factory가 이름과 일치하는 my_test 인스턴스를 생성합니다.

3

Build Phase

Top-Down 방식으로 하위 컴포넌트를 생성하여 계층 트리를 만듭니다.

4

Run Phase

Time-consuming phase가 시작되며 실제 검증이 수행됩니다.
