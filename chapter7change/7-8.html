Makefile 구조 분석

UVM 라이브러리는 시뮬레이터에 내장되어 있는 경우가 많습니다. VCS에서는 -ntb_opts uvm 옵션을 통해 자동으로 UVM 패키지와 매크로를 링킹합니다.

  • COMP_OPTS
    -sverilog (SystemVerilog 활성화), -timescale 등을 지정합니다.
  • UVM_OPTS
    UVM 라이브러리 경로 또는 내장 UVM 옵션을 지정합니다.
  • SIM_OPTS
    GUI 실행(-gui), UVM Test 이름 지정 등 실행 시간(Run-time) 옵션입니다.
# VCS Makefile Example
TESTNAME ?= base_test
SEED     ?= 1

VCS_CMD = vcs -sverilog -ntb_opts uvm-1.2 \
          -timescale=1ns/1ps \
          -debug_access+all \
          -f filelist.f

SIM_CMD = ./simv +UVM_TESTNAME=$(TESTNAME) \
          +ntb_random_seed=$(SEED)

comp:
	@echo "Starting Compilation..."
	$(VCS_CMD) | tee comp.log

sim:
	@echo "Running Simulation ($(TESTNAME))..."
	$(SIM_CMD) | tee sim.log

all: comp sim

+UVM_TESTNAME

컴파일 시 지정된 여러 Test 클래스 중, 어떤 테스트를 run_test()에서 인스턴스화하여 실행할지 결정합니다.

+UVM_VERBOSITY

출력되는 로그의 상세도를 조절합니다. LOW, MEDIUM, HIGH, FULL 등이 있습니다.

+UVM_OBJECTION_TRACE

시뮬레이션 종료 시점을 결정하는 Objection의 raise/drop 발생 기록을 추적하여 행잉(Hanging) 이슈 디버깅에 활용합니다.

💡 Transaction Recording 켜기

시뮬레이터가 Transaction 블록을 기록하게 하려면, 코드(Base Test 등)에서 recording_detail 설정을 켜야 합니다. 

function void build_phase(uvm_phase phase);
  super.build_phase(phase);
  // Enable TR recording for all components
  uvm_config_db#(int)::set(this, "*", 
    "recording_detail", UVM_FULL);
endfunction

Verdi UVM 디버깅 팁

  • RTL 신호선과 UVM Transaction 라인을 상하로 배치하여 타이밍을 직관적으로 비교하세요.

  • Sequence가 Item을 생성하여 Driver로 전달하는 시간적 흐름(Stream)을 파형에서 추적할 수 있습니다.
