Phase 7

결과 확인, 예외 처리 및 디버깅

UVM 환경에서 트랜잭션을 비교하고, 기능 커버리지를 측정하며, 예외 상황(Error Injection)을 주입하는 핵심 검증 메커니즘을 학습합니다.

7.1 UVM Report 시스템

UVM은 표준 $display 대신 강력한 메시지 필터링, 포맷팅, 파일 출력 기능을 제공하는 Report 시스템을 사용합니다. 시뮬레이션 단계별로 출력할 메시지의 양(Verbosity)을 커맨드 라인에서 제어할 수 있어 효율적인 디버깅이 가능합니다.

Report 매크로와 Verbosity

`uvm_info(ID, MSG, VERBOSITY)
일반적인 정보 출력. Verbosity 설정에 따라 출력 여부가 결정됨.
`uvm_warning(ID, MSG)
잠재적 문제 경고. 기본적으로 항상 출력됨.
`uvm_error(ID, MSG)
오류 발생. 카운트가 누적되며 한계치(quit_count) 도달 시 시뮬레이션 종료.
`uvm_fatal(ID, MSG)
치명적 오류. 즉시 시뮬레이션을 중단(Finish)함.

7.4 Callback 메커니즘

재사용성을 위해 Driver 나 Monitor 코드를 직접 수정하지 않고, 외부 Test 계층에서 uvm_callback을 이용하여 특정 시점에 개입(Hooking)할 수 있습니다. 주로 Error Injection (에러 주입) 테스트에 활용됩니다.

1. Driver에 매크로 삽입

task run_phase(...);
  forever begin
    seq_item_port.get_next_item(req);
    `uvm_do_callbacks(my_drv, my_cb, pre_send(req))
    drive_bus(req);
    seq_item_port.item_done();
  end
endtask

2. Test에서 Callback 등록 (Override)

function void build_phase(...);
  // Error 발생용 cb 객체 생성
  err_cb = err_inject_cb::type_id::create("err_cb");
  
  // 특정 Driver 인스턴스에 Callback 연결
  uvm_callbacks#(my_drv, my_cb)::add(env.agt.drv, err_cb);
endfunction
