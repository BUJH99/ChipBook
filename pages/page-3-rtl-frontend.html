Definition, Role, Output

Definition

Verilog, VHDL, SystemVerilog 기반 하드웨어 구조·동작 기술 언어, 소프트웨어 코드 형태를 빌린 회로 표현 수단

Role

대규모 회로의 코드 기반 설계·검증·합성 연결, RTL 수준 구조와 데이터 이동의 실제 설계 언어화

Output

Logic Synthesis

Definition, Role, Output

Definition

RTL을 Standard Cell Library에 매핑해 Gate-level Netlist로 변환하는 과정

Role

공정 구현 가능한 논리 형태 확보, 성능·전력·면적(PPA) 목표 달성의 출발점 형성

Output

Netlist, timing report, area/power report, 이후 DFT·PD에서 사용하는 구현 기반 데이터

장점 1

병렬 개발이 쉬워집니다.

장점 2

재사용성과 디버깅 효율이 올라갑니다.

장점 3

대형 칩을 manageable한 블록으로 나눌 수 있습니다.

SoC 정의

CPU, 메모리, 인터페이스, 보안, 전력관리 등 시스템 기능 블록을 하나의 칩 안에 통합한 구조입니다.

구성 요소

연산 블록, 메모리 계층, 고속 인터페이스, NoC, 보안 블록, 전력/클럭 관리 블록이 대표적입니다.

STA는 논리 기능을 시뮬레이션하는 것이 아니라, 최악 경로(Critical Path)의 지연을 계산해 주어진 클럭 주기 안에 데이터가 도착하는지 보는 분석입니다.

🐢
왜 최악 경로를 보나

가장 느린 경로 기준 timing 안전성 확인, 더 짧은 경로의 상대적 안전성 판단 기준

⏰
Setup 위반

데이터의 늦은 도착, 주기 완화 또는 경로 최적화 필요

⚡
Hold 위반

데이터의 너무 빠른 변화, 버퍼 삽입 등 지연 추가 필요

🧪
기능 검증과 차이

STA는 시간 분석, DV는 논리 기능 검증, 대체가 아닌 상호 보완 관계

Question
❓
Setup time vs Hold time 중 어느 것이 더 중요한가?

시험에서는 절대 비교보다 두 위반의 의미와 수정 방향을 구분하는지가 더 중요합니다.

✅
Answer
결론: 둘 다 중요

Setup은 목표 주파수와 성능에 직접 연결되고, Hold는 작은 위반이라도 남기면 안 되는 필수 수정 대상입니다.

1. 단순 연산
조합 로직 (Combinational)

LUT 직접 출력에 가장 잘 맞는 기본 Boolean 연산 패턴

LUT 
assign y = (a & b) | c;
실제 FPGA 매칭
CLB / LUT Direct Out

비교기, MUX, decode, 간단한 제어식의 LUT 중심 구현

2. 단순 저장
순차 로직 (Sequential)

플립플롭에 값 하나를 저장하는 가장 기본적인 FF 패턴

D-FF 
always @(posedge clk)
          q <= d;
실제 FPGA 매칭
CLB / D-FF Capture

레지스터, 상태 저장, 지연선 등 가장 기본적인 clocked storage 구조

3. 가장 효율적인 구조
파이프라인 (연산 + 저장)

LUT 뒤에 바로 FF를 붙여 timing을 가장 쉽게 잡는 대표 구조

LUT + FF 
always @(posedge clk)
          reg_out <= (a ? b : c);
실제 FPGA 매칭
CLB / LUT + FF Pipeline

속도와 라우팅 안정성을 동시에 얻기 좋은 FPGA 친화 패턴

4. 고속 산술 연산
DSP 블록 매핑

곱셈과 누산 형태는 LUT보다 DSP Slice로 보내는 것이 정석

DSP Slice 
always @(posedge clk)
          p <= (a * b) + c;
실제 FPGA 매칭
DSP Slice / Mult + Add

MAC, FIR, matrix, dot-product 계열에서 fmax와 전력 모두 유리

5. 대용량 메모리
BRAM 매핑

깊은 배열과 동기식 read/write는 Block RAM으로 유도하는 핵심 패턴

BRAM 
always @(posedge clk)
          if (we) ram[addr] <= din;
실제 FPGA 매칭
Block RAM / Sync Write

FIFO, line buffer, table memory, cache-like storage에 빈출

Resource Optimization
라우팅 및 자원 최적화 기법
6 - 14
6. 고속 덧셈 배선
Carry Chain

`+` 연산을 LUT 일반망이 아니라 전용 carry 망으로 태우는 구조

Carry 
assign sum = a + b;
Carry Chain
LUT0 / Sum[0] ⇣ LUT1 / Sum[1] ⇣ LUT2 / Sum[2] → 전용 Carry Out
실제 FPGA 매칭
CLB Carry Chain

Adder, accumulator, counter, compare에서 LUT 일반 경로보다 훨씬 빠른 구조

7. FF 자원 최적화
SRL (Shift Register LUT)

여러 단계 shift register를 FF 묶음 대신 LUT 내부 SRL 모드로 축소

SRL 
always @(posedge clk)
          shift_reg <= {shift_reg[14:0], din};
SRL 구조
din clk → LUT (SRL 모드) → D0…D15 → dout
실제 FPGA 매칭
LUT in SRL Mode

딜레이 라인, FIR tap, delay matching, serializer pre-stage에 유리

8. 글로벌 클럭망 보호
Clock Enable (CE) 매핑

Gated clock 대신 CE 핀을 쓰는 것이 FPGA 기본 원칙

CE 
always @(posedge clk)
          if (en) q <= d;
Clock Enable 매핑
Global clk en (Enable) → D-FF / CE → Q
실제 FPGA 매칭
FF CE Pin Mapping

Skew 폭증을 부르는 gated clock 회피, 글로벌 클럭망 보존 핵심

9. 비동기 초소형 메모리
LUTRAM (Distributed RAM)

작은 테이블 메모리를 LUT 기반 분산 RAM으로 처리하는 패턴

LUTRAM 
assign dout = lutram[r_addr];
Distributed RAM
Write Addr Data In WE → LUT Read Addr (async) → Data Out
실제 FPGA 매칭
Distributed RAM / LUTRAM

작은 lookup table, config RAM, score table 등에 빠른 비동기 read 제공

10. 외부 인터페이스 마진 확보
IOB Register Packing

출력/입력 레지스터를 I/O Block에 붙여 핀 타이밍을 안정화

IOB 
(* IOB = "true" *) reg data_out;
IOB Register Packing
FPGA Fabric → I/O Block (IOB) → D-FF → 외부 핀
실제 FPGA 매칭
IOB Packed Register

라우팅 지터 감소, 고정된 tco 확보, DDR/고속 출력 인터페이스에 중요

11. 제어 신호 최적화
동기식 리셋 (Sync Reset)

비동기 리셋 남용보다 sync reset이 FPGA hard block 활용에 유리

Reset 
always @(posedge clk)
          if (rst) q <= '0;
          else     q <= d;
Reset Strategy
Async rst ✕ Sync SR → D-FF → DSP / BRAM 호환
실제 FPGA 매칭
Sync Reset Friendly FF

DSP/BRAM 내부 FF 사용성, 라우팅 단순화, reset tree 부담 감소 효과

12. 상태 머신 가속
FSM One-Hot 인코딩

FF를 더 쓰더라도 decode depth를 줄여 fmax를 올리는 FPGA식 선택

FSM 
(* fsm_encoding = "one-hot" *)
        reg [3:0] state;
FSM Encoding
Binary → Deep LUT vs One-Hot → Shallow Logic
실제 FPGA 매칭
FF Rich / One-Hot Decode

복잡한 FSM일수록 LUT depth 절감과 timing 개선 효과가 크게 드러남

13. 비동기 클럭 안정성
CDC & ASYNC_REG

툴이 동기화 레지스터를 묶어서 배치하게 만드는 CDC 기본 속성

CDC 
(* ASYNC_REG = "true" *) reg sync1, sync2;
Clock Domain Crossing
TX Domain → FF1 → FF2 → RX Domain
실제 FPGA 매칭
2-FF Synchronizer

메타스테빌리티 완화, 툴의 packing/placement 유도, CDC 기본 습관

14. 글로벌 병목 해소
High Fanout 레지스터 복제

한 소스가 수천 개를 몰고 다니면 fanout 복제가 timing과 congestion을 구함

Fanout 
(* max_fanout = 50 *) reg global_en;
Logic Duplication
Src → Dup0 Dup1 Dup2 → Region Loads
실제 FPGA 매칭
Regional Register Duplication

enable, reset, control pulse, shared config 신호의 라우팅 병목 완화에 효과적

Physical Control
심화 물리 통제 기법
15 - 20
15. 딜레이 자동 밸런싱
레지스터 리타이밍 (Retiming)

조합 로직 앞뒤의 레지스터를 툴이 밀고 당겨 병목을 균등화

Retiming 
# Tool hint
        set_property STEPS.SYNTH_DESIGN.ARGS.RETIMING true [get_runs synth_1]
Register Balancing
Deep Logic 4ns → FF ⇢ Logic 2.5ns FF Logic 2.5ns
실제 FPGA 매칭
Before / After Retiming

주파수 한계가 특정 stage 한 곳에 몰릴 때 가장 강력한 자동 완화 기법

16. 하드 IP 극한 활용
DSP / BRAM 내장 파이프라인

패브릭 라우팅 대신 hard block 내부 레지스터를 최대한 살리는 고속화

A_REG / M_REG / P_REG 
// DSP / BRAM internal pipeline enable
        // A_REG, M_REG, P_REG all active
DSP Pipeline
A_REG → × → M_REG → + → P_REG
실제 FPGA 매칭
Hard IP Internal Pipeline

500MHz+ 같은 고속 타깃에서는 외부 FF보다 하드 블록 내부 FF 활용이 핵심

17. 로직 트리 재구성
Late Signal 병합 (섀넌 전개)

가장 늦는 제어 신호를 마지막 MUX 선택선으로 빼내는 구조적 최적화

Late MUX 
wire fast_path_a = ...;
        wire fast_path_b = ...;
        assign y = late_sig ? fast_path_b : fast_path_a;
Late Arriving Signal Optimization
병렬 선계산 A 병렬 선계산 B → 2:1 MUX late_sig
실제 FPGA 매칭
Parallel Precompute + Final MUX

late control signal이 병목일 때 logic depth를 드라마틱하게 줄이는 기법

18. 글리치 원천 차단
다중 비트 CDC (Gray Code)

멀티비트 포인터는 한 번에 1비트만 바뀌는 Gray Code가 기본

Gray Code 
assign gray = bin ^ (bin >> 1);
Binary vs Gray
Binary 011 → 100 3 bit Gray 010 → 110 1 bit
실제 FPGA 매칭
Pointer CDC / Gray Code

FIFO pointer, async counter crossing에서 메타스테빌리티 리스크를 크게 줄임

19. SDC 우선순위 구속
Timing Exceptions (False Path)

중요하지 않은 경로를 timing engine에서 빼서 critical path에 자원을 집중

SDC 
set_false_path -from [get_cells cfg_reg*] \
                       -to   [get_cells core_data*]
Timing Exceptions
Config Reg ⇢ False Path / Critical Path → 우선 최적화
실제 FPGA 매칭
Constraint Guided Optimization

진짜 중요한 데이터 경로에 타이밍 엔진과 optimizer의 힘을 몰아주는 제어 수단

20. 물리적 다이 경계 통제
SLR Crossing (Floorplanning)

멀티-SLR FPGA에서 die 경계를 넘는 경로에 파이프라인을 박는 기법

Pblock 
create_pblock slr_pipe
        add_cells_to_pblock [get_pblocks slr_pipe] [get_cells pipe_reg*]
SLR Crossing
Die 0 → Pipeline Reg → Die 1 → Pblock 제어
실제 FPGA 매칭
Multi-SLR Boundary Control

초대형 FPGA에서 가장 무서운 die-crossing delay를 설계 단계에서 미리 차단
