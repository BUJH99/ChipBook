Design Flow with Role

6-1. Role Map
6-2

Foundation IP

공정/라이브러리 기반
정의
역할

최첨단 반도체 공정을 바탕으로 Foundation IP를 설계·개발하는 영역입니다.

핵심 포인트

Standard Cell Library와 Embedded Memory처럼 모든 설계팀이 공통으로 활용하는 기반 자산을 만듭니다.

주요 업무
  • Standard Cell Library 설계와 PPA 최적화
  • Embedded Memory 및 SRAM 계열 매크로·컴파일러 개발
  • 공정 특성화와 라이브러리 검증, 설계팀 배포

Flow Stage. RTL 설계에 앞서 공통 설계 자산을 준비하며, 이후 합성·STA·PD 전반의 기준이 되는 기반을 제공합니다.

Standard Cell Embedded Memory Characterization Foundation IP
6-3

DP (Design Platform)

아키텍처 / 상위 설계
정의
역할

CPU와 NoC를 중심으로 시스템 반도체용 SoC Design Platform을 구축하는 영역입니다.

핵심 포인트

HDL 기반 RTL 설계부터 버스 구조 설계, IP 통합까지 이어지며 설계 효율을 높이는 플랫폼을 만듭니다.

주요 업무
  • HDL 기반 RTL 레벨 정밀 설계
  • CPU·NoC 기반 Bus 아키텍처 설계
  • 다양한 반도체 IP 통합과 SoC 플랫폼 구현
  • 고성능 연산·메모리 시스템 구성을 위한 상위 구조 설계

Flow Stage. 시스템 요구사항을 RTL과 플랫폼 구조로 구체화하는 상위 설계 단계의 핵심 역할을 담당합니다.

RTL CPU / NoC Bus Architecture SoC Platform
6-4

DV (Design Verification)

기능 검증
정의
역할

SoC와 IP가 의도한 기능을 정확히 수행하는지 검증해 설계 신뢰성을 확보하는 영역입니다.

핵심 포인트

고도화된 검증 환경과 커버리지 분석을 통해 제품 품질을 끌어올립니다.

주요 업무
  • SoC·IP 레벨 기능 검증과 시나리오 정교화
  • HVL/UVM 기반 검증 환경 및 자동화 체계 구축
  • 코드·기능 커버리지 분석과 커버리지 클로저
  • 버그 재현, 원인 분석, 수정 후 재검증

Flow Stage. RTL 완성 이후 합성에 들어가기 전까지, 설계가 의도한 대로 동작하는지를 집중적으로 검증하는 단계입니다.

UVM HVL Coverage IP / SoC Verification
6-5

DFT (Design For Test)

양산 테스트
정의
역할

ASIC과 SoC의 양산 품질을 확보할 수 있도록 테스트 구조와 전략을 설계하는 영역입니다.

핵심 포인트

Fault model을 바탕으로 테스트 가능성을 높이고 양산 경쟁력을 끌어올립니다.

주요 업무
  • Fault model 기반 Test Strategy 수립 및 구조 설계
  • SCAN, MBIST 등 양산형 테스트 구조 구현
  • 표준 기반 인터페이스 테스트와 ATPG 최적화
  • LBIST, POST 등 차세대 DFT 기술 적용

Flow Stage. 합성 전후의 넷리스트 단계에서 테스트 로직을 반영해, 양산 검사와 수율 확보를 준비하는 역할입니다.

SCAN MBIST ATPG LBIST / POST
6-6

PI (Physical Implementation)

합성 / 제약 / 저전력
정의
역할

논리 합성부터 타이밍 방법론, 저전력 구조, 사인오프 준비까지 연결해 물리 구현 가능성을 높이는 영역입니다.

핵심 포인트

개발 SoC 사양에 맞는 논리 레벨 구현과 구현 친화적 제약 조건을 완성합니다.

주요 업무
  • 논리 합성과 정밀 검증을 통한 최적 논리 레벨 구현
  • 복합 클럭 구조를 고려한 Timing Methodology와 제약 설계
  • UPF 기반 Dynamic/Static Power 관리와 저전력 구조 구현
  • 제품군별 Process·사인오프 Methodology 최적화

Flow Stage. RTL과 DV 결과를 바탕으로 물리 구현이 가능한 넷리스트와 제약 조건을 정리해 PD 단계로 넘기는 연결 단계입니다.

Synthesis Timing Low Power Signoff Methodology
6-7

PD (Physical Design)

Back-end
정의
역할

Floorplan부터 Routing, 사인오프까지 Physical Design 전 과정을 수행해 실제 칩 레이아웃을 완성하는 영역입니다.

핵심 포인트

최종적으로 신뢰성 높은 SoC를 테이프아웃 가능한 상태로 끌어올립니다.

주요 업무
  • Floorplan, Placement, CTS, Routing 등 Full-Flow Physical 구현
  • 공정 및 제품 특성에 맞춘 PPA 중심 Physical Design Flow 최적화
  • PG Grid, 저전력 Physical 설계, 고성능 CTS 구조 고도화
  • Physical Verification, SI/PI/EM 분석, ECO와 사인오프 클로저

Flow Stage. PI에서 전달된 넷리스트를 실제 레이아웃과 GDSII로 구현하고, 최종 물리 검증까지 수행하는 마지막 하드웨어 단계입니다.

Full-Flow PnR PPA Physical Verification SI / PI / EM
6-8

AP (Application Platform)

시스템 소프트웨어
정의
역할

설계된 칩이 실제 제품에서 동작하도록 펌웨어, BSP, 드라이버, 보안 환경을 구현하는 영역입니다.

핵심 포인트

하드웨어 설계 결과를 실제로 부팅되고 제어되는 시스템으로 완성합니다.

주요 업무
  • Cortex-M/A 기반 Bare-Metal 펌웨어와 부트 코드 개발
  • Secure Boot, ARM SystemReady, Trusted Firmware 기반 보안 환경 구축
  • Linux Kernel BSP와 Device Driver 개발, Automotive 환경 최적화
  • PCIe, DDR, Ethernet, USB 3.0 등 고속 인터페이스 검증과 보드 디버깅
Flow Stage

실리콘 브링업 이후 하드웨어 플랫폼 위에서 시스템 소프트웨어를 구동하고, 실제 제품 동작 환경을 완성하는 단계입니다.

Bare-Metal Secure Boot Linux BSP / Driver PCIe / DDR / USB3.0
6-9

함께 알아두면 좋은 연관 직무

인프라 / 패키징 / 세부 IP

정의 : 설계 인프라·패키징·Foundation IP 관련 역할

DT

EDA Infra

정의. 설계팀이 툴과 서버를 안정적으로 쓰도록 EDA 실행 환경과 자동화 인프라를 구축하는 역할입니다.

주요 업무
  • EDA 툴 셋업과 라이선스 운영
  • 서버, 배치 환경, 설계 farm 관리
  • 반복 업무 자동화와 flow 지원

Flow 위치. 특정 한 단계보다 모든 설계팀을 가로지르는 공통 인프라 축입니다.

EDA Tool License Server Farm Automation

CS

Chiplet / Package

정의. Chiplet, Si-interposer, HBM, 고속 인터페이스 SI/PI까지 다루는 첨단 패키징 및 시스템 연결 축입니다.

주요 업무
  • Multi-die / chiplet 구조 기획
  • Si-interposer, package 연동 검토
  • 고속 채널 SI/PI와 인터페이스 분석

Flow 위치. PD 이후 패키지·시스템 축으로 이어지며, HPC/AI 플랫폼 문맥과 자주 연결됩니다.

Chiplet Si-Interposer HBM SI / PI

SC

Foundation IP

정의. Standard Cell Library 개발과 특성화를 맡는 Foundation IP 세부 직무로, 합성과 STA의 출발점을 만듭니다.

주요 업무
  • 기본 logic cell 설계와 PPA 최적화
  • timing / power / area 특성화
  • library release와 공정 버전 관리

Flow 위치. RTL보다 앞에서 공통 라이브러리를 만들며 synthesis·STA·PD의 바닥을 형성합니다.

Standard Cell Characterization Liberty PPA

MC

Memory Compiler

정의. SRAM Compiler와 memory macro, 주변 회로, 옵션 특성화를 담당하는 Foundation IP 세부 직무입니다.

주요 업무
  • SRAM macro / compiler 구조 설계
  • redundancy, low-power 옵션 제공
  • 메모리 주변 회로와 특성화 지원

Flow 위치. SC와 함께 Foundation IP 축을 이루며, RTL 이전에 재사용 가능한 메모리 자산을 제공합니다.

SRAM Compiler Memory Macro Redundancy Low Power
6-10

심화 키워드 정리

Test / Firmware Keywords
DFT
MBIST

Memory Built-In Self Test. SRAM 같은 내장 메모리를 칩 내부 로직으로 직접 검사해 불량을 찾는 구조입니다.

DFT
BIRA / BISR

Built-In Redundancy Analysis / Repair. MBIST가 찾은 메모리 불량 위치를 분석하고 spare row/column으로 복구하는 메모리 수율 향상 기법입니다.

DFT
IJTAG

IEEE 1687 기반 내부 계측 접근 표준입니다. 칩 내부의 센서, 테스트 포인트, MBIST 제어 블록을 외부에서 구조적으로 접근하게 해줍니다.

DFT
LBIST

Logic Built-In Self Test. 메모리가 아니라 조합/순차 로직 쪽을 칩 내부 패턴 발생기와 시그니처 분석기로 검사하는 자체 테스트 방식입니다.

DFT
ATPG

Automatic Test Pattern Generation. stuck-at, transition 같은 결함 모델을 기준으로 양산 테스트용 패턴을 자동 생성하는 단계입니다.

AP
Linux BSP

Board Support Package. 특정 SoC와 보드에서 Linux가 부팅되도록 커널 설정, 드라이버, device tree, 초기화 코드를 묶은 시스템 소프트웨어 기반입니다.

AP
Bare-Metal Firmware

운영체제 없이 CPU가 전원 인가 직후 바로 실행하는 초기 펌웨어입니다. 부트 시퀀스, 클럭/메모리 초기화, 주변장치 bring-up, 보드 디버깅의 출발점으로 자주 연결됩니다.
