Part 11

AI Accelerator

Compute Architecture

AI Accelerator는 행렬 연산을 빠르고 효율적으로 처리하도록 특화된 연산 블록입니다

CPU가 범용 제어에 강하고 GPU가 대량 병렬 처리에 강하다면, AI accelerator는 MAC array, local SRAM, dataflow, quantization을 통해 딥러닝 추론 workload를 전력 효율 좋게 밀어내는 데 집중합니다. 면접에서는 연산량보다 memory movement가 더 큰 병목이라는 관점을 같이 설명하면 좋습니다.

MAC / SRAM / NPU / TOPS/W
Core Job
Matrix Multiply

conv, GEMM, attention 같은 반복 연산을 배열형 연산기로 빠르게 실행합니다.

Real Bottleneck
Memory Traffic

연산기 자체보다 weight, activation, partial sum을 어떻게 재사용하느냐가 성능과 전력을 좌우합니다.

Design Angle
Dataflow Choice

weight stationary, output stationary, row stationary 같은 선택이 아키텍처 색깔을 만듭니다.

11-1. Accelerator Stack

AI Accelerator를 블록 단위로 보면 이렇게 나뉩니다

Host / Scheduler / Compute Array / Local Memory / DRAM
11-2. Dataflow

자주 나오는 dataflow 키워드

Weight Stationary

weight를 local에 오래 두고 activation을 흘려보냅니다. weight 재사용이 큰 모델에 유리합니다.

Output Stationary

partial sum을 local에서 오래 유지해 accumulation write-back을 줄입니다.

Row Stationary

input, weight, partial sum 재사용을 균형 있게 맞추는 방향입니다.
