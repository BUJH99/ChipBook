10-2. Solution Thinking

solution 관점에서 자주 묻는 질문

이 제품의 병목은 어디인가?

compute가 느린지, memory bandwidth가 모자란지, bus contention이 심한지 먼저 분리해서 봅니다.

HW와 SW 경계는 어디서 맞물리나?

register map, interrupt, DMA descriptor, driver API가 실제 제품 완성도를 좌우합니다.

확장성이 있는가?

1개의 accelerator에서 4개로 커졌을 때 bus와 memory 구조가 버틸 수 있는지까지 봐야 합니다.
