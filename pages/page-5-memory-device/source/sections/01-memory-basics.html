Deep Dive Analysis

DRAM 및 SRAM 불량 메커니즘 심층 분석

반도체 메모리 칩은 나노미터(nm) 단위의 미세 공정이 적용됨에 따라 다양한 물리적, 전기적 요인으로 인해 오류가 발생할 수 있습니다. 본 섹션에서는 메모리 반도체의 핵심인 DRAM과 SRAM에서 발생하는 주요 불량 메커니즘을 엔지니어링 및 아키텍처 관점에서 상세히 분석합니다.

D

DRAM Fail Mechanisms

DRAM은 커패시터(Capacitor)에 전하(Charge)를 축적하여 데이터를 저장하는 방식을 취합니다. 셀의 집적도가 높아질수록 주변 회로와의 간섭 및 미세한 공정 산포에 민감해지며, 엄격한 타이밍 마진 부족으로 인한 데이터 유실이 발생할 수 있습니다.

Static/Dynamic
① 리텐션 불량 (Retention Fail)

원인: 커패시터나 셀 트랜지스터의 누설 전류(GIDL, Junction Leakage). 온도가 높을수록 급증합니다.

결과: 리프레시 주기가 오기 전에 전하가 소실되어 데이터 에러 발생.

개선: High-k 절연막 도입, 온도 센서 연동 가변 리프레시(ATCSR).

Destructive Read
② 리스토어 페일 (Restore Fail)

원인: 파괴적 읽기 후 센스 앰프가 값을 셀에 되돌려 써야 하는데, 구동력 저하 또는 워드라인 유지 시간(tRAS) 부족으로 복원이 끝까지 진행되지 못함.

결과: 읽는 순간 거의 비워진 셀이 충분히 재충전되지 못해, 다음 접근 전까지 약한 전하 상태로 남아 데이터 유실 위험이 커짐.

개선: 센스 앰프 구동력 강화, tRAS 타이밍 마진 확보.

Offset / Mismatch
③ 센스 앰프 마진 불량

원인: 공정 산포로 인한 센스 앰프 TR 간 미스매치 또는 커플링 노이즈 유입.

결과: 미세한 ΔV를 반대로 증폭하여 1을 0으로 오판독(Read Fail).

개선: 레이아웃 대칭성 확보, Twisted Bitline 적용.

Coupling Effect
④ 로우 해머링 (Row Hammer)

원인: 인접 Aggressor Row(WL)를 짧은 시간에 반복 활성화할 때 발생하는 전자기적 간섭.

결과: 인접 행(Victim) 셀의 전하 유실 및 Bit Flip.

개선: 타겟 행 인접부를 선제적 리프레시하는 TRR(Target Row Refresh) 도입.

Timer / tRFC
⑤ 리프레시 페일

원인: 리프레시 타이머 정밀도 저하 또는 할당된 시간(tRFC) 내 100% 충전 실패.

결과: 전하량이 부족한 셀(Weak Cell)부터 리텐션 불량이 조기 발생합니다.

대책: 타이머 보정 회로 강화, 불량 셀을 예비 셀로 교체(Spare Cell Repair).

Write Timing
⑥ 라이트 리커버리 페일

원인: 신규 데이터 기입 후 전압이 안정화될 대기 시간(tWR)이 부족할 때 발생합니다.

결과: 새로운 데이터가 완전히 충전되기 전에 워드라인이 닫혀 Write Fail이 납니다.

대책: Write Driver 구동력 강화, tWR 타이밍 엄격 보장.

Intermittent Fail
⑦ 가변 보존 시간 (VRT)

원인: TR 산화막 내부 결함(Trap)으로 인해 전자가 불규칙하게 포획 및 방출됩니다.

결과: 같은 셀도 테스트 시점마다 누설 전류와 보존 시간이 달라져 간헐적으로 리텐션 불량이 발생합니다.

대책: 스트레스 조건(고온/고전압)의 Burn-in Test를 통해 사전 스크리닝.

Physical / Radiation
⑧ 하드 디펙트 & 소프트 에러

원인: 파티클에 의한 금속 단락/단선(Hard), 우주 방사선 충돌(Soft).

결과: Hard는 회로가 영구적으로 망가지고, Soft는 회로는 정상인데 데이터만 순간 반전(SEU)됩니다.

대책: 예비 회로 Laser Repair 적용(Hard), On-Die ECC 탑재(Soft).

S

SRAM Fail Mechanisms

SRAM은 6개의 트랜지스터(6T)가 상호 대칭적인 래치(Latch) 구조를 이루어 작동합니다. 초고속 성능이 요구되는 환경이므로 찰나의 타이밍과 내부 노드 간의 미세한 전압 균형이 정상 동작의 핵심입니다.

Read Destructive
① 읽기 오류 (Read Upset)

원인: 읽기 직전 BL/BLB는 모두 precharge되어 있고, WL이 켜지면 저장된 0 노드(Q)가 BL에 연결되어 위로 끌려 올라갑니다.

결과: SNM이 부족해 Q가 inverter trip point를 넘으면 읽기 동작이 파괴적으로 변해 셀 데이터가 0/1에서 1/0으로 반전됩니다.

개선: WL Underdrive, Read Buffer 분리형 8T SRAM, 셀 비율(Cell Ratio) 강화.

Write Margin
② 쓰기 실패 (Write Fail)

원인: BL/BLB가 새 값을 밀어 넣어도 cross-coupled inverter의 복원력이 더 커서 내부 노드가 trip point를 넘지 못합니다.

결과: WL이 꺼지는 순간 셀이 다시 기존 0/1 상태로 돌아가므로, 쓰기 동작을 했어도 최종 저장은 실패합니다.

개선: Write Assist (셀 VDD 일시 강하, Boosted WL, Negative BL)로 래치 우위를 잠시 약화시킵니다.

Array Level
③ 하프 셀렉트 디스터브

원인: 타겟 열만 write를 걸어도 같은 행의 WL이 켜지는 동안 옆 열 셀의 access path도 함께 열립니다. 이때 옆 열 bitline이 1/1 precharge 상태면 읽기와 비슷한 전하 교환이 생겨 half-select stress가 걸립니다.

결과: 저장된 0 노드가 조금씩 들리며 SNM이 줄고, 반복 write나 저전압 조건에서는 취약 셀만 disturb upset으로 이어질 수 있습니다.

개선: 8T처럼 read path를 분리한 셀, column interleaving, write assist tuning, array 배치 최적화로 half-select stress를 줄입니다.

Vmin Limit
④ 대기 전력 불량 (Hold Fail)

원인: 대기 전력을 줄이려고 standby VDD를 너무 낮추면 cross-coupled inverter의 복원력이 약해져, 내부 누설과 오프셋을 더 이상 이기지 못합니다.

결과: 읽기나 쓰기를 하지 않아도 저장된 0/1 차이가 서서히 무너지며, 취약 셀은 idle 상태에서 데이터가 소실됩니다.

개선: Data Retention Voltage(DRV) 하한을 PVT별로 측정하고, retention guardband와 hold assist, 스크리닝으로 안전 여유를 확보합니다.

Timing Violation
⑤ 접근 시간 지연 (Slow Bit)

원인: 국부적인 공정 산포나 배선 저항(RC) 증가로 인한 트랜지스터 스위칭 속도 저하.

결과: 정해진 클럭 내에 응답하지 못해 시스템 타이밍 위반 및 오작동 발생.

대책: Speed Binning 테스트 적용, 제품 동작 클럭 다운그레이드.

Mismatch & Noise
⑥ 공정 산포와 노이즈 (RTN)

원인: TR 간 문턱전압 미스매치 및 전자의 무작위 포획/방출(Random Telegraph Noise).

결과: 정적인 미스매치로 이미 기운 래치가 RTN에 의해 순간적으로 더 흔들리면서 read / hold SNM이 크게 줄어듭니다.

대책: 채널 통제력이 뛰어난 3D FinFET 및 GAA 구조 전면 도입.

HCI / NBTI / EM
⑦ 소자 열화 (Aging)

원인: 장기간 구동 스트레스(HCI, NBTI)로 인한 문턱전압 상승 및 금속 배선 단선(EM).

결과: 제품 사용 시간이 길어짐에 따라 점진적 성능 저하 및 프리징 현상 발생.

대책: 수명 연한 모델링(Aging Simulation), 설계 시 충분한 Guard Band 확보.

Radiation Effect
⑧ 소프트 에러 (SER)

원인: 알파 입자 등 고에너지 입자가 셀 내부 래치를 타격하여 전자쌍이 생성됨.

결과: 물리적 파괴는 없으나 데이터가 반전되어 시스템 크래시(Kernel Panic 등) 직결.

대책: 아키텍처 레벨의 물리적 Interleaving 및 강력한 시스템 레벨 ECC 적용.
