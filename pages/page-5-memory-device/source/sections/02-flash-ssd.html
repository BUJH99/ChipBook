NAND - Program / Read / Erase

NAND는 전하를 가두면 문턱전압(Vth)이 올라가고, 읽을 때는 전류가 흐르는지 여부로 데이터를 판별합니다.

초기 상태는 erase 예시입니다. trapped charge가 거의 없어 문턱전압이 낮고, read 전압에서 채널 전류가 흘러 1로 감지됩니다.

Example SLC convention: Erase = low Vth = current flows = 1, Program = trapped charge = high Vth = blocked current = 0.

01  NAND Flash Cell
  • Floating gate에 전하를 저장하여 0/1을 표현
  • 수십 ~ 수백 개의 cell이 모여 page 구성
  • p-Si substrate 위에 적층 구조 형성
02  Flash Package / Die
  • Page: 읽기/쓰기의 최소 단위
  • Block: 삭제(erase)의 최소 단위
  • Spare(OOB) 영역으로 신뢰성 확보
03  Controller + Firmware
  • FTL이 Logical ↔ Physical 주소 변환
  • ECC로 데이터 오류 검출 및 정정
  • Wear Leveling으로 수명 균일화
04  SSD (System)
  • 고성능, 고신뢰성, 대용량 저장장치 제공
  • 호스트 인터페이스로 시스템과 연결
  • 사용 가능한 용량(Usable Capacity) 제공

Controller + Firmware
FTL · ECC · Wear Leveling
Management Layer

FTL: host의 논리 주소를 실제 flash 물리 주소로 바꿔주는 계층

ECC: 저장 중 생기는 bit error를 검출하고 정정하는 회로/코드

Wear Leveling: 특정 block만 빨리 닳지 않게 erase/write를 분산하는 기법

가장 넓은 sensing margin을 확보하는 기본 형태입니다. charge 차이가 커서 속도와 endurance 쪽이 유리합니다.

Fast Read / Program High Endurance
MLC
2 bit / cell
4 states

한 셀을 4개 영역으로 나눠 density를 높입니다. read reference가 늘어나고 SLC보다 더 세밀한 Vth 구분이 필요합니다.

Balanced Density Tighter Read Margin
TLC
3 bit / cell
8 states

8개 상태를 구분해야 하므로 program step과 read calibration이 더 정교해집니다. consumer SSD에서 널리 쓰이는 절충형입니다.

Mainstream SSD More Calibration
QLC
4 bit / cell
16 states

16개 상태를 잘라 써야 해서 density는 가장 높지만, sensing window는 가장 촘촘합니다. ECC와 read retry의 부담이 커지는 영역입니다.
