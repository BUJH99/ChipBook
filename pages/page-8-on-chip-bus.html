Address Map 해석 예시

주소 범위는 단순한 메모리 배치가 아니라, 어떤 Slave로 트랜잭션을 라우팅할지 결정하는 하드웨어 규칙입니다.

주소 범위 선택 대상 용도
0x0000_0000 - 0x0FFF_FFFF SRAM / ROM 명령어 및 데이터의 저지연 접근 영역
0x1000_0000 - 0x1FFF_FFFF AHB Peripheral DMA 등 고속 제어 블록 연결 영역
0x4000_0000 - 0x4000_FFFF APB Peripheral UART, GPIO, Timer의 레지스터 접근 영역

실제 SoC에서는 decoder가 이 memory map에 따라 대상 Slave를 선택하므로, 주소 범위는 곧 버스 라우팅 규칙을 의미합니다.

AMBA 버스를 APB / AHB / AXI로 분화하는 이유

하나의 BUS 구조로 모든 요구사항을 만족시키기 어렵기 때문에, AMBA는 대역폭, 복잡도, 전력, 동시성 요구에 따라 프로토콜을 계층적으로 분화했습니다.

APB

레지스터 접근 중심의 저속 주변장치는 구조의 단순성, 구현 용이성, 저전력 특성이 우선되므로 APB가 적합합니다.

AHB

중간 수준의 대역폭과 공유형 시스템 버스가 필요한 구간에서는 burst와 pipeline을 지원하는 AHB 구조가 효율적입니다.

AXI

CPU, DMA, DDR처럼 높은 처리량과 동시 전송이 필요한 구간에서는 독립 채널 기반의 AXI 구조가 필요합니다.

APB (Advanced Peripheral Bus)

  • - 저전력과 설계 단순화 중심의 버스 구조
  • - 단일 전송 기반의 순차적 통신 방식

        AXI (Advanced eXtensible Interface)

        • - 5개 독립 채널 기반의 고대역폭 인터페이스 구조
        • - 주소·데이터 분리와 동시 read/write 처리 구조
        • - Out-of-Order 및 outstanding transaction 지원 구조

        AXI Channel Architecture Diagram

        Figure 1-1 Channel architecture of AXI

        쓰기 동작을 위한 3개 채널과 읽기 동작을 위한 2개 채널이 완전히 독립적으로 존재해 양방향 동시 전송이 가능합니다.

        AXI VALID / READY Handshake Diagram

        Figure 1-2 Fundamental Handshake

        AXI의 모든 5개 채널은 동일한 VALID / READY 핸드셰이크 메커니즘을 사용하여 정보(주소, 데이터, 응답)를 전송합니다.

        • VALID 신호: 정보를 보내는 측(Source)에서 구동하며, 채널의 주소나 데이터가 유효함을 나타냅니다.
        • READY 신호: 정보를 받는 측(Destination)에서 구동하며, 데이터를 받아들일 준비가 완료되었음을 나타냅니다.
        • 전송 발생 조건: VALID와 READY가 모두 HIGH인 클럭 상승 에지에서 전송이 이루어집니다.
