On-Chip Bus

8-1. AMBA Protocol

SoC 속의 도로망,
AMBA 프로토콜 이해하기

스마트폰의 두뇌인 SoC(System on Chip) 내부에서 CPU, 메모리, 주변기기들이 데이터를 주고받는 규칙, AMBA와 on-chip bus 구조를 5개 목차로 정리합니다.

AMBA가 무엇인가요?

AMBA(Advanced Microcontroller Bus Architecture)는 ARM에서 정의한 온칩(On-Chip) 버스 프로토콜 집합입니다. 하나의 SoC 안에서 CPU, DMA, 메모리, 각종 주변장치가 같은 규칙으로 통신하게 해주는 표준 도로망이라고 이해하면 쉽습니다.

SoC 내부 AMBA 버스 구조도

APB

저속 주변장치용 (타이머, UART, GPIO)

AHB

고속 모듈용 (SRAM, DMA, Flash)

BUS (공통 개념)

BUS는 단순한 선 묶음이 아니라 주소(Address), 데이터(Data), 제어(Control)를 정해진 타이밍에 교환하는 통신 규칙입니다. 누가 요청하는지(Master), 누가 응답하는지(Slave), 언제 전송이 끝나는지(Ready / Response)를 함께 봐야 구조가 이해됩니다.

BUS 기본 데이터 흐름

Figure 1-1 Generic Bus Communication Model

모든 bus는 요청자와 응답자, 그리고 중간에서 주소를 해석하고 연결하는 interconnect 관점으로 읽으면 구조가 단순해집니다.

BUS를 볼 때 먼저 확인할 것

Master / Slave 관계

누가 요청을 시작하고 누가 응답하는지부터 정리하면 신호 방향이 쉬워집니다.

Address / Data / Control 분리

주소, 실제 데이터, 제어 정보는 역할이 다르므로 별도 신호군으로 봐야 합니다.

Ready / Response

전송 완료와 대기, 에러를 어떻게 표현하는지가 protocol 차이의 핵심입니다.

Address Map 예시

Address Range Target Role
0x0000_0000 - 0x0FFF_FFFF SRAM / ROM 빠른 instruction / data access
0x1000_0000 - 0x1FFF_FFFF AHB Peripheral DMA, high-speed control blocks
0x4000_0000 - 0x4000_FFFF APB Peripheral UART, GPIO, Timer register map

실제 SoC에서는 decoder가 이 memory map을 보고 어떤 slave를 선택할지 결정합니다.

BUS를 APB / AHB / AXI로 나누는 이유

APB

설정값을 읽고 쓰는 저속 peripheral은 단순함과 저전력이 더 중요합니다.

AHB

중간 대역폭과 공유형 backbone이 필요할 때 pipeline과 burst가 유리합니다.

AXI

CPU, DMA, DDR처럼 동시성과 높은 throughput이 필요한 경로는 채널 분리가 필요합니다.

APB (Advanced Peripheral Bus)

데이터 전송 속도보다는 전력 소모를 줄이고 설계를 단순화하는 데 목적이 있는 버스입니다. 신호등이 없는 한적한 시골길이나 골목길처럼, 한 번에 하나씩 천천히 안전하게 통신합니다.

APB Master-Slave Interface Diagram (공식 문서 규격)

Figure 2-1 APB Master-Slave Interface

APB 브릿지(Master)와 주변장치(Slave) 간의 물리적 신호 연결 다이어그램입니다. 공식 문서의 학술적 스타일(흑백, 직각 선)을 반영했습니다.

AHB (Advanced High-performance Bus)

시스템 성능 향상을 위해 파이프라인(Pipelining) 기법과 버스트(Burst) 전송을 지원하는 고속 버스입니다. 여러 개의 마스터(Multiple Masters)가 버스를 공유할 수 있으므로, 버스 사용 권한을 제어하는 Arbiter(중재자)가 필수적으로 존재합니다.

AHB Architecture Diagram (Master, Slave, Arbiter)

Figure 3-1 AHB Architecture Block Diagram

AHB 시스템의 기본 구성요소인 마스터, 슬레이브, 아비터, 그리고 이들을 연결하는 상호연결망의 데이터 흐름입니다.

AHB Transfer Timing Diagram (WaveDrom)

AHB 파이프라이닝 분석:
- Address Phase와 Data Phase가 서로 교차하며 실행되는 파이프라인 구조입니다.
- 주소 HADDR가 전달된 다음 클럭에 HWDATA를 쓰거나 HRDATA를 읽습니다.
- INCR4 vs WRAP4: INCR는 주소가 계속 증가하고, WRAP은 특정 경계에 도달하면 시작 주소로 되돌아갑니다.
- Wait State: 슬레이브가 HREADY를 LOW로 구동하면 현재 Data Phase와 다음 Address Phase가 함께 연장됩니다.

AXI (Advanced eXtensible Interface)

현재 가장 고성능을 자랑하는 AMBA 프로토콜입니다. 5개의 독립적인 채널을 사용해 주소와 데이터를 분리하고, 읽기와 쓰기를 동시에 처리할 수 있습니다. 또한 비순차적(Out-of-Order) 전송과 outstanding transaction을 지원해 시스템 효율을 극대화합니다.

AXI Channel Architecture Diagram

Figure 1-1 Channel architecture of AXI

쓰기 동작을 위한 3개 채널과 읽기 동작을 위한 2개 채널이 완전히 독립적으로 존재해 양방향 동시 전송이 가능합니다.

AXI VALID / READY Handshake Diagram

Figure 1-2 Fundamental Handshake

AXI의 모든 5개 채널은 동일한 VALID / READY 핸드셰이크 메커니즘을 사용하여 정보(주소, 데이터, 응답)를 전송합니다.

  • VALID 신호: 정보를 보내는 측(Source)에서 구동하며, 채널의 주소나 데이터가 유효함을 나타냅니다.
  • READY 신호: 정보를 받는 측(Destination)에서 구동하며, 데이터를 받아들일 준비가 완료되었음을 나타냅니다.
  • 전송 발생 조건: VALID와 READY가 모두 HIGH인 클럭 상승 에지에서 전송이 이루어집니다.
