Part 9

Off-Chip Interface

Chip Boundary

Off-Chip Interface는 칩 바깥 세상과 연결되는 관문입니다

메모리, 저장장치, 보드, 다른 칩과 통신하려면 외부 인터페이스가 필요합니다. 이 영역은 단순 protocol 이름 암기보다 controller + PHY + signal integrity + timing margin 관점으로 설명할 수 있어야 실무 감각이 살아납니다.

DDR / PCIe / USB / SerDes
Boundary Role
외부 장치 연결

DDR 메모리, SSD, 카메라, 다른 SoC, PC host 같은 외부 장치와 실제로 신호를 주고받습니다.

Key Layer
Controller + PHY

상위 protocol 처리 logic와 전기적 신호를 담당하는 PHY를 분리해서 보는 습관이 중요합니다.

Physical Risk
SI / PI / Timing

패키지, 보드, 채널 손실, jitter, skew, termination 조건까지 함께 묶어서 봐야 합니다.

9-1. Interface Stack

Protocol 위에 logic이, 아래에는 PHY가 있습니다

Software / Protocol / Controller / PHY / Package / Board
9-2. Common Interfaces

자주 나오는 외부 인터페이스

DDR / LPDDR

메모리 인터페이스. controller, PHY, training, timing margin, DQ/DQS 관계가 핵심입니다.

PCIe

고속 serial interconnect. lane 수, link training, LTSSM, equalization, throughput가 포인트입니다.

USB / UFS / Ethernet

외부 장치, 저장장치, 네트워크 연결 축입니다. protocol stack과 PHY 조건을 함께 봐야 합니다.

9-3. Debug Checklist

문제가 났을 때 어디를 의심하나

Protocol Violation

packet format, state transition 오류

Training Fail

link up 실패, calibration 실패

Clock / Reset Sequence

bring-up 순서 이상

Signal Integrity

jitter, skew, eye margin 부족

Power / Termination

I/O 전압, impedance, termination 설정

Package / Board Path

chip 밖 채널 손실과 routing 영향
