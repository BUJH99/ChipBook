AXI Lite Channel Architecture Diagram

Figure 1-1 Channel architecture of AXI Lite

AXI Lite도 5개 독립 채널을 유지하지만, 각 read/write transaction은 1-beat 단일 전송으로 제한됩니다.

AXI Lite VALID / READY Handshake Diagram

Figure 1-2 Fundamental Handshake

AXI Lite의 모든 5개 채널은 동일한 VALID / READY 핸드셰이크 메커니즘을 사용하여 주소, 데이터, 응답을 전송합니다.

  • VALID 신호: 정보를 보내는 측(Source)에서 구동하며, 채널의 주소나 데이터가 유효함을 나타냅니다.
  • READY 신호: 정보를 받는 측(Destination)에서 구동하며, 데이터를 받아들일 준비가 완료되었음을 나타냅니다.
  • 전송 발생 조건: VALID와 READY가 모두 HIGH인 클럭 상승 에지에서 전송이 이루어집니다.

    AXI Lite Transaction Dependencies

    Write Path

    Write transaction dependencies

    Master-driven Slave-driven

    핵심 규칙: BVALID는 AWVALID/AWREADY와 WVALID/WREADY의 두 핸드셰이크가 모두 끝난 뒤에만 올라올 수 있습니다.

    • Master 측: AWVALID, WVALID는 각 READY를 기다리지 않고 독립적으로 구동해야 합니다.
    • Slave 측: 단순 AXI Lite slave는 AWVALID와 WVALID가 모두 준비된 시점에 AWREADY/WREADY를 함께 여는 경우가 많습니다.
    • Response: BREADY는 BVALID보다 먼저 올라와도 되고 나중에 올라와도 됩니다. 둘 다 HIGH일 때 write response가 완료됩니다.

    ARM IHI 0022H Figure A3-7 기준으로, double-headed arrow는 해당 신호가 앞선 조건이 성립한 뒤에만 assert될 수 있음을 의미합니다. 또한 BVALID는 마지막 write data beat, 즉 WLAST까지 확인된 뒤에만 생성될 수 있습니다.

    Read Path

    Read transaction dependencies

    Master-driven Slave-driven

    핵심 규칙: RVALID는 읽기 주소 핸드셰이크(ARVALID/ARREADY)가 먼저 완료되어야만 생성될 수 있습니다.

    • Master 측: ARVALID는 ARREADY를 기다리지 않고 먼저 구동해야 합니다.
    • Slave 측: ARREADY는 내부 준비 상태에 따라 지연될 수 있고, 주소를 수락한 뒤 RVALID로 데이터를 반환합니다.
    • Data Return: RREADY 역시 먼저 HIGH로 유지될 수 있으며, RVALID와 동시에 HIGH가 되는 시점에 read data transfer가 완료됩니다.

    AXI Lite Signal Descriptions

    Signal Width Source Description
    Global Signals
    ACLK, ARESETn 1bit / 1bit System AXI Lite 글로벌 클럭과 리셋 신호입니다. 모든 전송은 ACLK 상승 에지에서 샘플링됩니다.
    Write Address Channel (AW)
    AWADDR [31:0] Master 쓰기할 제어 레지스터 주소를 지정합니다. AXI Lite에서는 단일 주소 접근만 처리합니다.
    AWVALID / AWREADY 1bit / 1bit Master / Slave 쓰기 주소 채널의 핸드셰이크 신호입니다.
    Write Data Channel (W)
    WDATA [31:0] Master 지정한 레지스터에 기록할 데이터입니다.
    WSTRB [3:0] Master 바이트 단위 write enable입니다. 어떤 바이트 lane이 유효한지 지정합니다.
    WVALID / WREADY 1bit / 1bit Master / Slave 쓰기 데이터 채널의 핸드셰이크 신호입니다.
    Write Response Channel (B)
    BRESP [1:0] Slave 쓰기 결과가 OKAY인지 ERROR인지 응답합니다.
    BVALID / BREADY 1bit / 1bit Slave / Master 쓰기 응답 채널의 핸드셰이크 신호입니다.
    Read Address Channel (AR)
    ARADDR [31:0] Master 읽을 제어 레지스터 주소를 지정합니다.
    ARVALID / ARREADY 1bit / 1bit Master / Slave 읽기 주소 채널의 핸드셰이크 신호입니다.
    Read Data Channel (R)
    RDATA [31:0] Slave 레지스터에서 읽어온 데이터입니다.
    RRESP [1:0] Slave 읽기 결과의 상태 응답입니다.
    RVALID / RREADY 1bit / 1bit Slave / Master 읽기 데이터 채널의 핸드셰이크 신호입니다.
