APB (Advanced Peripheral Bus)

  • - 저전력과 설계 단순화 중심의 버스 구조
  • - 단일 전송 기반의 순차적 통신 방식

APB Master-Slave Interface Diagram

Figure 2-1 APB Master-Slave Interface

APB 브릿지(Master)와 주변장치(Slave) 간의 물리적 신호 연결 다이어그램입니다. 공식 문서의 학술적 스타일(흑백, 직각 선)을 반영했습니다.

Table 4-1 APB Signal Descriptions

Signal Source Description
PCLK Clock source Clock. APB의 모든 전송은 PCLK의 상승 에지(rising edge)에 동기화됩니다.
PRESETn System bus equivalent Reset. Active LOW 리셋 신호입니다. 일반적으로 시스템 버스 리셋과 직접 연결됩니다.
PADDR APB bridge Address. 주변장치 버스 브릿지가 구동하는 주소 버스입니다. (최대 32 bits)
PSELx APB bridge Select. 슬레이브 장치가 선택되었음을 알립니다. 슬레이브마다 각각 독립적인 PSELx 신호가 존재합니다.
PENABLE APB bridge Enable. APB 전송의 두 번째 사이클(ACCESS 단계) 및 그 이후를 나타내는 신호입니다.
PWRITE APB bridge Direction. HIGH일 때 쓰기(Write) 액세스를, LOW일 때 읽기(Read) 액세스를 의미합니다.
PWDATA APB bridge Write data. 쓰기 사이클 동안 브릿지가 구동하는 쓰기 데이터 버스입니다. (최대 32 bits)
PREADY Slave interface Ready. 슬레이브가 처리를 완료했는지(HIGH), 혹은 전송 사이클을 연장할지(LOW) 결정하는 신호입니다.
PRDATA Slave interface Read Data. 읽기 사이클 동안 선택된 슬레이브가 구동하는 읽기 데이터 버스입니다. (최대 32 bits)
PSLVERR Slave interface Transfer Error. 전송 실패를 나타냅니다. 지원하지 않는 주변장치의 경우 해당 핀은 LOW로 고정됩니다.
