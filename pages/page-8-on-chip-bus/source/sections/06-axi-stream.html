AXI STREAM (Advanced eXtensible Interface Stream)

Stream
Addressless Data Path
  • 주소 없는 payload 전달
  • sample / pixel / packet 연속 흐름
  • 영상 · 오디오 · DSP pipeline 적합
Flow Control
VALID / READY
  • TVALID / TREADY handshake
  • TREADY 기반 backpressure
  • stall 구간의 현재 beat 유지
Packet
Frame Boundary
  • TLAST 기반 packet / frame 경계
  • TKEEP / TSTRB 기반 byte 유효 범위
  • 마지막 beat 정렬 정보

AXI STREAM Signal Descriptions

Signal Width Source Description
Global Signals
ACLK 1bit System AXI Stream 전송 기준 클럭입니다. 모든 beat는 ACLK 상승 에지에서 샘플링됩니다.
ARESETn 1bit System Active LOW reset입니다. stream channel 상태와 내부 packet counter를 초기화합니다.
Core Handshake
TDATA [DATA_W-1:0] Source 실제 payload 데이터입니다. sample, pixel, packet body가 이 버스로 전달됩니다.
TVALID 1bit Source 현재 beat의 데이터와 sideband가 유효함을 알립니다.
TREADY 1bit Sink sink가 다음 beat를 받을 준비가 되었음을 알리는 backpressure 신호입니다.
Byte Qualifiers
TKEEP [DATA_W/8-1:0] Source 각 byte lane의 유효 여부를 표시합니다. 마지막 beat에서 부분 유효 바이트를 설명할 때 자주 사용됩니다.
TSTRB [DATA_W/8-1:0] Source 데이터 바이트와 위치 바이트를 구분하는 strobe입니다. 구현에 따라 생략되는 경우도 많습니다.
TLAST 1bit Source 현재 beat가 packet 또는 frame의 마지막 beat임을 표시합니다.
Routing / Metadata
TID [ID_W-1:0] Source stream ID를 나타내는 optional 신호입니다. 다중 flow를 구분할 때 사용합니다.
TDEST [DEST_W-1:0] Source packet의 목적지나 routing hint를 담는 optional 신호입니다.
TUSER [USER_W-1:0] Source application-specific metadata를 담는 사용자 정의 sideband입니다. SOF, error flag, color info 등을 실을 수 있습니다.
