10-1. Integration Stack
10-1. Integration Stack

SoC Solution 통합 맵

Solution은 여러 블록과 소프트웨어가 한 제품의 목표를 달성하도록 통합되는 구조입니다.
문제 정의부터 하드웨어, 버스, 메모리, 소프트웨어, 검증까지 하나의 유기적인 흐름으로 연결해야 완벽한 실제 제품이 됩니다.

System Architecture Flow
Goal

Product Goal (제품 목표)

단순한 개별 성능 수치가 아닌, 최종 제품이 목표로 하는 정량적 기준(성능, 전력 소모, 칩 면적, 총 제조 비용)을 먼저 엄격히 정의합니다. 제품의 타겟 시장(AI 서버, 모바일, 차량용 등)에 따라 요구되는 PPA 우선순위가 결정되며, 이를 바탕으로 모든 IP의 선택과 커스텀 아키텍처 설계를 개시합니다.

Fit

Architecture Fit (구조 정합)

하드웨어 연산기(Compute)와 메모리 대역폭(Memory), 온칩 버스(Bus Interconnect), 외부 I/O 포트 사이의 성능 병목(Bottleneck)을 분석하여 최적의 균형점을 맞춥니다. 한 쪽이 아무리 빨라도 다른 한 쪽이 느리면 시스템 전체 속도가 고사하는 현상을 아키텍처 매칭과 정합 최적화를 통해 해결합니다.

Path

Bring-up Path (실리콘 활성화)

설계된 칩이 물리적인 보드 위에서 깨어나 오동작 없이 원활하게 구동하기까지의 경로를 설계 단계에서 함께 수립합니다. 레지스터 맵 설계, 인터럽트 라우팅, 부트 시퀀스 구성, 디바이스 드라이버 및 실리콘 검증 소프트웨어까지 하드웨어와 소프트웨어가 유기적으로 맞닿도록 연결고리를 만듭니다.

10-2. System Bottleneck & HW/SW Co-Design

시스템 병목 해결과 HW/SW 설계 경계

반도체 칩이 진정한 솔루션이 되려면 개별 IP의 우수성을 넘어 전체 시스템 레벨의 병목을 해결하고,
하드웨어와 소프트웨어가 밀결합되는 제어 인터페이스(Control Interface)가 극도로 정밀하게 정의되어야 합니다.

Performance Boundary

1) Roofline Model을 통한 시스템 제약 분석

시스템 성능은 산술 연산 밀도(Arithmetic Intensity)에 따라 메모리 대역폭(Memory Bandwidth)에 지배받는 Memory-Bound 구간과 하드웨어 최고 연산 능력에 지배받는 Compute-Bound 구간으로 나뉩니다.

Key Strategy

연산 강도가 낮은 설계(A)는 아무리 CPU/NPU 코어 클럭을 높여도 DRAM 속도 한계(Memory Wall)에 갇힙니다. 로컬 SRAM 캐시 최적화, 데이터 타일링(Data Tiling) 기법을 통해 연산당 메모리 접근률을 최대한 줄여 곡선의 꺾임점 우측(B)으로 시스템 동작 지점을 이동시키는 것이 솔루션 설계의 핵심입니다.

Interface Pipeline

2) 하드웨어와 소프트웨어의 제어 및 통신 루프

드라이버 소프트웨어와 하드웨어 연산 가속기가 레지스터 맵(MMIO), DMA 전송, 그리고 완료 보고용 인터럽트(Interrupt) 구조를 통해 어떻게 고속 데이터 제어 주기를 완수하는지 보여줍니다.

Key Strategy

CPU가 가속기에 고성능 연산을 요청할 때 매번 메모리를 읽는 것은 극도로 비효율적입니다. 따라서 CPU는 레지스터에 가속기가 처리할 주소와 길이만 기록하고(1), 가속기 내부의 고속 DMA 엔진(2)이 시스템 버스를 장악해 메모리에서 연산기로 데이터를 전달한 뒤, 연산이 끝나면 인터럽트(3)를 발생시켜 CPU가 인터럽트 핸들러(ISR)로 빠져나오게(4) 설계합니다.

C 실제 하드웨어 레지스터 맵 제어 C 드라이버 예시

accel_driver.c
// 1. NPU 하드웨어 가속기 메모리 맵 레지스터 정의
typedef struct {
    volatile uint32_t CTRL;         // 0x00: 제어 레지스터 (Bit 0: Start, Bit 1: Int Enable)
    volatile uint32_t DMA_SRC;     // 0x04: DMA 소스 주소 (DRAM 메모리 위치)
    volatile uint32_t DMA_DST;     // 0x08: DMA 목적지 주소 (SRAM 로컬 캐시 위치)
    volatile uint32_t DMA_LEN;     // 0x0C: DMA 전송 데이터 바이트 크기
    volatile uint32_t STATUS;      // 0x10: 상태 레지스터 (Bit 0: Busy, Bit 1: Int Triggered)
} NpuAccelRegs;

#define NPU_BASE_ADDR (0x40001000)
NpuAccelRegs* const npu = (NpuAccelRegs*)NPU_BASE_ADDR;

// 2. 드라이버 API: 가속기 구동 함수
void npu_start_transfer(uint32_t src_addr, uint32_t dst_addr, uint32_t len) {
    // (A) 가속기가 이전 연산을 완전히 마치고 유휴 상태(Idle)가 될 때까지 폴링
    while (npu->STATUS & 0x1);

    // (B) DMA 주소 및 길이 설정 (하드웨어 제어 경계)
    npu->DMA_SRC = src_addr;
    npu->DMA_DST = dst_addr;
    npu->DMA_LEN = len;

    // (C) 인터럽트를 허용하고 가속기 및 DMA 구동 트리거!
    npu->CTRL = (0x1 | 0x2); // Bit 0 = Start, Bit 1 = Interrupt Enable
}

// 3. 인터럽트 서비스 루틴 (ISR) - 가속기 연산 완료 시 자동 호출
void npu_interrupt_service_handler(void) {
    // (A) 인터럽트 유발 소스 확인
    if (npu->STATUS & 0x2) {
        // (B) 다음 전송 차단을 피하기 위해 인터럽트 클리어 플래그 기록
        npu->STATUS = 0x2;  // W1C (Write 1 to Clear)

        // (C) 드라이버 상위 계층 혹은 세마포어를 잠금 해제하여 프로세스 작업 복귀
        release_system_semaphore();
    }
}
10-3. Scalability & Verification Loop

시스템 확장성과 실리콘 검증 루프

솔루션의 지속 가능한 성장을 위한 다중 다이(Multi-Die) 칩렛(Chiplet) 아키텍처 확장 전략과,
천문학적인 실리콘 제작 실패 비용을 방지하기 위한 시뮬레이션 기반 시스템 검증 루프를 학습합니다.

Chiplet Expansion

1) Chiplet 및 Unified NoC 확장 모델

단일 다이 면적(Reticle Limit) 극복을 위해 연산 다이(Compute Die)와 입출력 다이(I/O Die)를 분리하고, UCIe 고속 인터커넥트와 Unified NoC를 묶어 단일 패키지 솔루션으로 결합합니다.

Validation Pipeline

2) Pre-to-Post Silicon 종합 검증 루프

칩 설계가 진행됨에 따라 가상 프로토타입부터 에뮬레이터, FPGA, 그리고 마침내 물리 칩의 디버깅 단계까지, 각 단계별 모델의 클럭 대역폭 속도와 상호 피드백 관계를 도출합니다.

★ 핵심 면접 답변 키워드 (Interview Hooks)

System Bottleneck & Memory Wall

가장 연산 속도가 느린 메모리 및 온칩 인터커넥트의 대역폭 한계가 칩의 최대 PPA 효율을 제한(Amdahl's Law)합니다. 가속기 아키텍처 면접 시, 캐시 활용을 극대화하여 연산 밀도(Arithmetic Intensity)를 꺾임점 우측으로 향상시켜 대역폭 벽을 우회했다는 점을 반드시 논리적으로 설명해야 합니다.

HW/SW Co-Design & Register Interface

하드웨어 가속기만 잘 만든다고 시스템 속도가 빨라지지 않습니다. 레지스터 맵(MMIO) 제어 인터페이스 설계, 순환 DMA 링 버퍼 디스크립터 구조, 인터럽트 레이턴시 정책이 소프트웨어 디바이스 드라이버와 유기적으로 물리도록 Co-Design 되어야 가속기의 물리 유효 대역폭이 극대화된다는 점을 설명하십시오.

MCM Chiplet Scalability & NoC

칩 내부 코어 수나 대역폭 확장 시 단일 다이(Die) 면적 초과로 수율이 하락하는 한계를 해결하기 위해, Compute Die와 I/O Die를 물리적으로 쪼개고 UCIe 고속 2.5D/3D 패키징과 단일 Unified NoC 기술로 연결해 전체 시스템 통합 성능을 유지한 확장성 설계 경험을 답변 키워드로 활용하십시오.

Validation Loop & Silicon Bring-up

칩 제작 실패 시의 손실을 방지하려면 pre-silicon 단계에서 SystemC 아키텍처 탐색, UVM 기반 기능 검증, 하드웨어 에뮬레이션, 실시간급 FPGA 보드 검증까지 촘촘한 검증 피드백 루프를 돌아야 합니다. 이후 실제 실리콘이 입고되면 JTAG 디버깅 및 OS 부트스트랩 완료까지의 파이프라인 지식을 어필하십시오.
