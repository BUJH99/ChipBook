Part 2

Design Flow

2-1. Design to Product Flow
STAGE 01

제품 정의 (Spec)

시장 타겟 PPA(Power, Performance, Area) 결정 및 시스템 아키텍처 수립

STAGE 02

논리 설계 (RTL)

마이크로 아키텍처 설계, Verilog/SystemVerilog 기반 코딩 및 IP 통합

STAGE 03

검증 & 합성

UVM/Formal 기반 기능 검증(DV) 후 Gate-level Netlist 변환 및 DFT 삽입

STAGE 04

물리 설계 (PD)

Floorplan, P&R, STA 튜닝, DRC/LVS 및 IR-Drop 검증 완료 후 GDS 확보

STAGE 05

Tape-out & Fab

Foundry로 설계 이관, 마스크 제작 및 웨이퍼 레벨 대량 생산 진행

STAGE 06

Post-Si 검증

양산된 실물 칩을 보드에 실장하여 S/W 연동 디버깅 및 시스템 신뢰성 최종 테스트

Floorplan

Initial Physical Planning

칩의 전체 크기와 윤곽을 정하고, I/O 핀, 매크로(SRAM, IP), 전원망의 큰 배치를 먼저 잡는 초기 physical planning 단계입니다. 이후 단계의 혼잡도와 timing 품질을 크게 좌우하는 출발점입니다.

Chip Size Macro Power Grid

Placement

Standard Cell Placement

합성된 수많은 standard cell을 timing, 배선 혼잡도, 면적 효율을 고려해 적절한 위치에 배치하는 단계입니다. 이 단계가 흔들리면 이후 CTS와 Routing에서 문제가 크게 증폭됩니다.

Standard Cell Congestion Density

CTS

Clock Tree Synthesis

클럭 신호가 칩 내부의 플립플롭에 균형 있게 도달하도록 트리 형태의 분배망을 만드는 작업입니다. 핵심은 skew를 줄이고, clock latency와 전력 부담을 함께 제어하는 데 있습니다.

Clock Tree Skew Latency

Routing

Metal Interconnect

배치된 셀과 매크로의 핀을 실제 금속층으로 연결해 전기적 경로를 완성하는 단계입니다. timing뿐 아니라 제조 규칙까지 만족해야 하므로 DRC와도 밀접하게 이어집니다.

Metal Layer Connectivity DRC

RC Extraction

Parasitic Extraction

실제 배선에서 생기는 저항(R)과 커패시턴스(C)를 추출해 후반 timing 분석의 정확도를 높이는 작업입니다. Routing 이후 timing이 갑자기 나빠지는 이유를 이해할 때 함께 봐야 하는 핵심 개념입니다.

Parasitic R Parasitic C Post-Route

STA

Static Timing Analysis

시뮬레이션 벡터 없이 회로의 모든 timing path를 수학적으로 분석해 setup과 hold가 만족되는지 확인하는 작업입니다. 합성 이후에도, 배치배선 이후에도 반복해서 수행됩니다.

Setup Hold Timing Path

LEC

Logic Equivalence Check

RTL과 합성 netlist, 또는 최적화 전후 netlist가 논리적으로 같은 기능을 유지하는지 비교 검증하는 절차입니다. 구조가 바뀌더라도 기능은 같아야 하므로 ECO나 최적화 이후에 특히 중요합니다.

RTL vs Netlist Equivalence ECO Check

ECO

Engineering Change Order

마지막 단계에서 발견된 timing 위반이나 논리 오류를 전체 flow를 처음부터 다시 돌리지 않고 국소적으로 수정하는 방식입니다.

Local Fix Timing Fix Signoff Loop

DRC

Design Rule Check

레이아웃이 파운드리 공정 규칙을 어기지 않았는지 확인하는 물리 검증입니다. 배선 폭, 간격, via 규칙처럼 제조 가능성과 직결된 항목을 검사하므로 Routing 이후 signoff의 핵심 축이 됩니다.

Foundry Rule Spacing Manufacturability

LVS

Layout Versus Schematic

작성된 레이아웃의 연결 관계가 원래 의도한 회로도나 netlist와 같은지 비교하는 검증입니다. DRC가 규칙 준수 확인이라면, LVS는 연결 정합성 검증이라고 구분하면 헷갈림이 줄어듭니다.

Layout Netlist Connectivity

IR Drop

Voltage Drop Analysis

전원 공급점에서 부하 블록까지 전류(I)가 흐르는 동안 전원선의 저항(R)이 누적되면서, 시작점보다 끝단의 실제 전압이 더 낮아지는 현상을 분석하는 signoff 검증입니다. 전류가 큰 블록이나 power grid가 약한 구간에서 심해지며, local VDD 부족은 timing 악화와 기능 불안정으로 이어질 수 있습니다.

Power Grid Current I V = I x R

EM

Electromigration

금속 배선에 높은 current density가 오래 걸리면, 전자 흐름 방향으로 금속 원자가 서서히 밀려 이동합니다. 그 결과 한쪽에는 금속이 비어 void가 생기고, 다른 쪽에는 금속이 쌓여 hillock이 생기며, 장기적으로는 배선 단선이나 신뢰성 저하로 이어질 수 있습니다.

Current Density Reliability Void / Hillock

Std Cell Library

Standard Cell Timing Library

합성과 physical design이 기준으로 삼는 기본 셀 집합입니다. 각 셀의 기능, 지연, 전력, 면적 정보가 담겨 있어 synthesis가 gate를 고르고, STA와 PD가 timing 품질을 계산하는 출발점이 됩니다.

Cell Function Delay Power / Area

SDC Constraint

Synopsys Design Constraints

클럭, 입력 지연, 출력 지연, false path, multicycle path 같은 timing 조건을 정의하는 제약 파일입니다. 회로 자체를 바꾸는 파일은 아니지만, synthesis와 STA가 회로를 어떤 기준으로 해석할지 정해주는 핵심 입력입니다.

Clock I/O Delay False Path

UPF

Unified Power Format

저전력 설계 의도를 기술하는 파일로, power domain, isolation, level shifter, retention 같은 구조를 정의합니다. RTL만으로는 드러나지 않는 전력 동작 규칙을 synthesis, PI, PD 단계에 전달하는 역할을 합니다.

Power Domain Isolation Level Shifter

Signoff Deck

Foundry Rule / Check Deck

파운드리가 제공하는 최종 검증 규칙 세트로, DRC, LVS, RC extraction 같은 signoff 툴이 무엇을 어떤 기준으로 검사할지 정의합니다. 실무에서는 “공식 검증 기준서”에 가까운 의미로 이해하면 됩니다.

Foundry Rule DRC / LVS Official Check

GDSII

Graphic Data System II

GDSII는 Graphic Data System II의 약자로, 최종 physical layout을 제조용으로 담아낸 대표 데이터 포맷입니다. Signoff를 통과한 레이아웃 결과가 이 파일로 정리되어 파운드리로 전달됩니다.

Tapeout Data Layout Polygon Foundry Handoff

Sanity Check

Quick Consistency Check

본격적인 full run을 돌리기 전에 파일 연결, constraints, library, netlist 상태가 기본적으로 맞는지 빠르게 확인하는 사전 점검입니다. 큰 문제를 초기에 걸러 시간을 아끼는 기본 건강검진으로 이해하면 됩니다.

Pre-Check Consistency Quick Validation
